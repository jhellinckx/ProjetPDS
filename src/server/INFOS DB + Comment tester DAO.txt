Pour tester les DAO, user ou food, il faut d'abord créer une database (je propose qu'on l'appelle tous db_appli) et deux tables différentes dont le code est donné ci-dessous:

create table db_appli.User (
id INT(11) not NULL auto_increment,
username VARCHAR(60) NOT NULL,
gender VARCHAR(20) NOT NULL,
primary key (id),
UNIQUE (username)
) ENGINE = INNODB;


create table db_appli.Food (
id INT(11) not NULL auto_increment,
url VARCHAR(60) NOT NULL,
code VARCHAR(60) NOT NULL,
productName VARCHAR(60) NOT NULL,
primary key (id)
) ENGINE = INNODB;


Une fois que vous avez créer ces tables (pour User on attend que raph découvre cmt implémenter les colonnes appreciatedFood & depreciatedFood ; pour food, well, c'est un exemple en attendant la table de openfoodfact de raph qui aura ses propres identifiants de colonnes & les infos qu'on retirera de chaque aliment (glucides, apport energétique tout ca tout ca).
Pour ajouter des des values à la tables pour les test , envoyez cette requete sql:

INSERT INTO User (username, gender) VALUES ('exempleUserName','male');

pour food:

INSERT INTO Food (url,code,productName) VALUES ('http://lol.com','IJZJ2R','banane');

Apres changer le user & mdp de votre user qui à acces a la db dans dao/dao.properties.

Ensuite il suffit de "make runtestDAO" pour qu'il trouve tout ce qu'il y a ds la db & les affiche.

Sinon checker testDAO.java & doTest pour voir comment utiliser les méthodes d'une DAO.
